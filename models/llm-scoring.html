Model Lab 03

LLM Feedback Scoring

← All demos
Unlike Labs 01–02, this isn't a regression. A large language model reads free text — a Google review, a survey verbatim, a maintenance ticket — and returns a fixed JSON structure: sentiment, topics, severity, churn signal, routing. The "math" is a contract: same prompt, same schema, temperature 0, every time. Those structured outputs are what feed the regressions in Labs 01–02 (e.g., the sentiment score is a feature in the renewal model). Full method: docs/llm-scoring-methodology.md.

The Prompt Contract

        
The exact instruction sent with every text — this IS the model spec


You are a scoring function for a rental-housing
operator. Read the resident text and return ONLY
valid JSON matching this schema:

{
 "sentiment": float −1.0 … +1.0,
 "topics": [up to 3 from fixed taxonomy:
   elevator, maintenance_speed, noise, staff,
   amenities, billing, cleanliness, parking,
   pests, safety, communication, rent_value,
   move_in_out, other],
 "severity": "low" | "medium" | "high",
 "churn_signal": true | false,
   // true only on explicit leaving intent
   // or unresolved repeat failure
 "routing": "maintenance" | "resident_services"
          | "property_manager" | "none",
 "summary": string, max 15 words
}

Rules: temperature 0. No prose outside JSON.
If text is ambiguous, severity="low" and
churn_signal=false (fail conservative).
