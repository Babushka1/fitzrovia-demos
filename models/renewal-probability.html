Model Lab 01

Renewal Probability · Logistic Regression

← All demos
What this model does: logistic regression takes a set of resident signals — tenure, rent increase offered, service tickets, sentiment, amenity engagement — assigns each a learned weight, combines them into a score, and squashes that score to a renewal probability between 0% and 100%. The weights are discovered by gradient descent: start all weights at zero, show the model thousands of past lease outcomes, nudge each weight in the direction that reduces prediction error, repeat until the error stops falling. In production, the inputs are real lease data from Yardi; the model structure is identical.

The Model Equation

A weighted sum squashed to a probability
// score: weighted sum of resident features
z = b0 + w1·tenure + w2·rentIncrease%
     + w3·tickets90d + w4·slaBreaches
     + w5·sentiment + w6·amenityUses
     + w7·isStudent

// sigmoid: squash score to 0–100%
P(renew) = 1 / (1 + e^(−z))

// training: for 400 epochs
//   error = P(renew) − actualOutcome
//   each weight -= learningRate · error · feature
How training works: the model starts with all weights at zero and repeats hundreds of times — predict every lease in the training set, compare against what actually happened, nudge each weight in the direction that reduces total error. The learned weights tell you which signals actually predict renewal in your specific portfolio; they are not assumed in advance.

What the weights mean

Each weight is a lever on renewal probability

A positive weight means that signal pushes a resident toward renewal — longer tenure, higher sentiment, and more amenity engagement all increase the score. A negative weight pushes toward churn — a large rent increase, SLA breaches on their tickets. The sigmoid function ensures no single signal can push the probability below 0% or above 100%, regardless of how extreme the input is.

What this enables in practice: for every resident in the portfolio, the model produces an explainable score — not a black box. Each feature's contribution can be surfaced alongside the probability, so a PM can see exactly why a resident is flagged as at-risk and which intervention — fixing an outstanding SLA breach, adjusting the renewal offer — moves the needle most. This is the engine behind the risk scores in the Service Recovery Workbench.
