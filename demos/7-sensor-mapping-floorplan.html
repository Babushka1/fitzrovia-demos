How the map gets built — commissioning

One afternoon per building, done once
1
Export the device list. The access-control system already has reader IDs; the WiFi controller lists APs; both usually have location names that are stale or vague ("DOOR_3F_02").
2
Walk-test. Walk the floor with a test fob and a phone: badge every reader, note timestamp → match to the log → confirm which physical door each ID is. For WiFi, stand in each zone and watch which AP your test device associates to.
3
Record coverage overlap. Where AP circles overlap (corridor bleed), assign a primary zone and discount rule — e.g., devices seen by AP-07 for <5 min are corridor pass-through, not lounge users.
4
Save as the mapping table. From then on, every raw event (reader R-14, 07:14, credential #8812) auto-resolves to "gym entry, 7am, resident of unit X (pseudonymized)."
5
Cross-validate sources. Booking says party room reserved 7–10pm; fob shows 1 entry; sensor shows 0 occupancy after 8pm → no-show detected. Disagreement between sources is itself signal.
